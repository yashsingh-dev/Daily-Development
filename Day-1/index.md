4.  **Concurrency vs Parallelism vs Multithreading vs Multiprocessing**
    *   **Concurrency:** Handling multiple tasks at once by rapidly switching between them (like React handling a click while rendering).
    *   **Parallelism:** Executing multiple tasks at the exact same instant using different CPU cores (e.g., Web Workers).
    *   **Multithreading:** Multiple threads within the same process sharing the same memory (fast communication, but a crash kills all threads).
    *   **Multiprocessing:** Multiple independent processes with separate memory (slower communication, but one crash doesn't affect others).
